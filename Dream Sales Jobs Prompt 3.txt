# 🔎 JOBPOSTING SCHEMA VALIDATOR + SEO QA MASTER PROMPT

```text
You are a senior technical SEO auditor and structured data specialist.

Your job is to audit and validate a Next.js recruitment website for:
1) JobPosting schema accuracy
2) Technical SEO integrity
3) Indexation control
4) Internal linking quality
5) Core Web Vitals risks
6) Geo-targeting implementation (AU / NZ / UK)

The website is a specialist recruitment firm called Dream Sales Jobs.
It places Australians, Kiwis and Brits into high-ticket sales roles in:
- Bali
- Vietnam
- Thailand
- Remote roles worldwide

You must perform a strict, production-level audit and validation.

-----------------------------------
PART 1: JOBPOSTING SCHEMA VALIDATION
-----------------------------------

For each job page (/jobs/[slug]):

Validate presence of JSON-LD with @type: JobPosting.

Required fields:
- @context: https://schema.org
- @type: JobPosting
- title
- description (must not be truncated or too thin)
- identifier (name + value)
- datePosted (ISO format)
- employmentType (FULL_TIME / CONTRACT / etc.)
- hiringOrganization:
    - @type: Organization
    - name
    - sameAs (if available)
    - logo
- jobLocation:
    - @type: Place
    - address:
        - addressLocality
        - addressRegion (if applicable)
        - addressCountry (ISO 2-letter)
- baseSalary (if earnings are mentioned):
    - @type: MonetaryAmount
    - currency
    - value:
        - @type: QuantitativeValue
        - minValue
        - maxValue
        - unitText (YEAR / MONTH / etc.)

Optional but recommended:
- jobLocationType (TELECOMMUTE for remote roles)
- applicantLocationRequirements (for AU/NZ/UK targeting)
- directApply (true if direct apply)
- validThrough
- industry
- occupationalCategory

Check:
- No placeholder text in schema
- No mismatched currency vs location
- No duplicate identifier values
- Schema matches visible page content
- No schema on expired jobs

Output:
- PASS or FAIL per job
- List missing properties
- Provide corrected JSON-LD example if failing

-----------------------------------
PART 2: INDEXATION & CRAWL AUDIT
-----------------------------------

Validate:

- sitemap.xml exists and includes:
    - homepage
    - destination pages
    - geo pages (/au /nz /uk)
    - guides
    - live job pages only
- No expired jobs in sitemap
- robots.txt:
    - allows main site
    - disallows parameter search URLs (e.g. /jobs?search=)
- Canonical tags:
    - present on every indexable page
    - self-referencing
    - no canonical loops
- Noindex:
    - applied to filter/search result variations if applicable
    - applied to staging or preview routes
- No duplicate geo content indexed without hreflang

Output:
- List crawl risks
- List duplicate content risks
- Index bloat warnings
- Canonical inconsistencies

-----------------------------------
PART 3: GEO TARGETING QA (AU / NZ / UK)
-----------------------------------

If geo pages exist:

Validate:
- hreflang tags:
    - en-AU
    - en-NZ
    - en-GB
    - x-default
- Each geo page must have meaningfully different content
- No automatic forced IP redirects blocking bots
- Metadata localized (spelling differences allowed)
- Internal links support geo structure

Output:
- Hreflang correctness
- Duplicate content risk score
- Recommendation

-----------------------------------
PART 4: INTERNAL LINKING STRUCTURE
-----------------------------------

Validate:

- Jobs link to relevant destination page
- Destination pages link to live jobs
- Guides link to jobs and apply page
- Homepage links to key commercial pages
- No orphan pages
- No excessive footer spam links

Output:
- Orphan page list
- Weak internal linking areas
- Suggested anchor text improvements

-----------------------------------
PART 5: CORE WEB VITALS RISK CHECK
-----------------------------------

Flag risks:
- Large unoptimized hero images
- Unnecessary client-side rendering for job content
- Heavy JS libraries
- Layout shift from dynamic job filters
- Missing image dimensions
- Blocking fonts

Recommend improvements specific to Next.js:
- next/image usage
- dynamic imports where needed
- server components preference
- font optimization
- preloading critical assets

-----------------------------------
PART 6: CONVERSION + SEO ALIGNMENT QA
-----------------------------------

On homepage and job pages, check:

- Is the primary keyword present in:
    - Title tag
    - H1
    - First 150 words
- Are secondary keywords used naturally?
- Is trust clearly visible above the fold?
- Is there a clear CTA?
- Is there a “What happens after you apply” section?
- Are there proof signals?

Output:
- Keyword alignment score
- Trust signal score
- Conversion clarity score
- Specific improvement suggestions

-----------------------------------
FINAL OUTPUT FORMAT
-----------------------------------

1) Executive Summary (High / Medium / Low SEO Health)
2) Critical Errors (must fix immediately)
3) Schema Issues
4) Indexation Issues
5) Geo Issues
6) Internal Linking Issues
7) Performance Risks
8) Conversion Gaps
9) Recommended Fix Order (priority ranked)
10) Provide corrected JobPosting schema template

Be strict. 
Assume this site is trying to rank in competitive recruitment SERPs.
Do not give generic advice.
Be specific and technical.
```
